Overview

VarViz displays multiple layers of protein-level annotation on a shared amino-acid axis so you can evaluate variant significance at a glance. All data is fetched live from public APIs - no pre-downloaded databases required beyond a gene-coordinate file and CCRS regions. VarViz integrates 20+ computational pathogenicity predictors via dbNSFP/MyVariant.info and auto-computes ACMG/AMP evidence tags per Richards et al. (2015) [11], Tavtigian et al. (2020) [12], and Pejaver et al. (2022) [13].

Figure 1. VarViz workflow architecture. User inputs are processed through eight live APIs; local files and clinical data feed the ACMG/AMP engine. Outputs include interactive plots, static figures, and a TSV export.

Core Features

  • VIZ 9 interactive tracks: gnomAD frequency, mutation density, AlphaFold pLDDT, AF Mean Pathogenicity, ClinVar/PTM/CCRS, Multi Conservation (UCSC), ConSurf, UniProt domains, transmembrane topology
  • VIZ Multi Conservation (UCSC) track: per-residue PhyloP 100V, PhyloP 470M (mammalian), and PhastCons 100V fetched live from the UCSC REST API, mapped to amino acid positions via Ensembl exon structure
  • VIZ Amino acid color-coding: lollipop stick = reference AA physicochemical group; dot = variant AA group; different colors = physicochemical property shift
  • TABLE Variant Annotation Summary: 40+ columns in 6 color-coded sections per variant
  • TABLE Hybrid ACMG classifier: Richards 2015 rule-based + Tavtigian 2020 Bayesian points + Pejaver et al. (2022) [13] PP3/BP4 calibration
  • TABLE AlphaMissense exact scores: per-variant pathogenicity from AlphaFold substitution CSV with MyVariant.info fallback
  • TABLE Layout: Variant Summary → Download TSV → External Links (15 databases)

Quick Start

  1. Select gene: type an HGNC symbol (e.g. TP53, BRCA1, DDR2). VarViz maps it to UniProt, GRCh38 coordinates, and protein length.
  2. Enter variants: paste comma-separated HGVS protein notation: p.R175H, p.G245S, p.V272M. Both 3-letter and 1-letter forms accepted.
  3. Choose tracks: tick desired Output Tracks. Multi Conservation (UCSC) and Conservation (ConSurf) are independent toggles.
  4. Upload optional files: ConSurf grades file (before Go), custom pathogenic variant positions.
  5. Click Go: VarViz fetches from 8+ live APIs with progress updates. Cached genes reload instantly.
  6. Explore: hover plots, scroll the Variant Summary, click Download TSV, switch to GeneInfo tab, use External Links.
Tip: All API responses are cached in-session. Returning to a previously loaded gene costs zero network calls.

Plot Tracks

1 - Protein Domain Bar (always on)

UniProt features at amino acid coordinates: functional domains, regions, transmembrane helices (orange), topological domains (extracellular/cytoplasmic/lumenal), active sites, binding sites, disulfide bonds, PTMs, signal peptides. User variants shown as color-coded lollipops at 65° with 6-level staggered labels.

2 - gnomAD Frequency

Rainfall plot of gnomAD v4 (GRCh38) population variants. Rare/absent = red, common = blue. Dashed red line marks the allele frequency cutoff. Auto-retries and falls back to gnomAD v2.1.1 (GRCh37) on failure.

3 - Mutation Density

Smoothed density curves of variant counts per position window. Blue = gnomAD, red = ClinVar pathogenic, green = user-uploaded pathogenic positions (optional).

4 - AF Mean Pathogenicity

Per-residue mean AlphaMissense pathogenicity averaged across all 19 possible amino acid substitutions at each position, from the AlphaFold substitution matrix CSV. Reflects intrinsic mutational tolerance independent of which specific variant is queried.

5 - ClinVar / PTM / CCRS

Three annotation layers: ClinVar pathogenic/LP variant positions (colored by significance), UniProt PTMs (phosphorylation, ubiquitination, etc.), and CCRS constrained coding region boundaries. CCRS ≥90th percentile zones are used by the PM1 ACMG rule.

6 - Multi Conservation (UCSC) NEW

Three conservation score lines from the UCSC Genome Browser REST API (hg38), mapped to amino acid positions via Ensembl canonical transcript exon structure:

  • PhyloP 100V: 100-vertebrate alignment. Positive = conserved, negative = evolving faster than neutral
  • PhyloP 470M: 470-mammalian alignment, finer mammalian resolution
  • PhastCons 100V: probability of falling in a conserved element (0–1)

These scores feed directly into the PP3/BP4 ACMG conservation gate (see Criteria Details).

7 - Conservation (ConSurf)

Per-residue evolutionary conservation grade (1–9) and normalized score from a user-uploaded ConSurf grades file. Grade 1 = most variable; grade 9 = most conserved. Burial status (B/E) shown when available. The track appears automatically when a file is uploaded. There is no checkbox. Upload consurf.grades before clicking Go.

8 - AlphaFold pLDDT

Per-residue structural confidence (0–100). Color-coded: Very High 90–100 (dark blue), High 70–90 (light blue), Low 50–70 (yellow), Very Low <50 (orange). Shown in Variant Info as the pLDDT column with category label.

Per-Variant Cosegregation

Each variant card has its own Cosegregation dropdown. Select the number of affected family members with the variant to apply PP1:

  • Not assessed: PP1 not applied (default)
  • 1-2 affected relatives: PP1 Supporting (+1 pt)
  • 3-4 relatives: PP1 Moderate (+2 pts)
  • 5+ relatives: PP1 Strong (+4 pts)

The classification updates when you change the selection. Cosegregation choices are saved in the TSV download as Cosegregation, PP1_Applied, Final_ACMG_Tags, Final_Classification, and Final_Points columns.

Note: Different variants in the same gene can segregate in different families. Setting cosegregation per card avoids applying the same PP1 level to all variants at once.

ACMG Classification - Overview

VarViz implements a hybrid two-tier ACMG/AMP classification engine:

  1. Rule-based classifier (Richards et al. 2015): fires when evidence tag combinations match a defined rule (e.g. “1 PS + 1 PM = Likely Pathogenic”)
  2. Bayesian point fallback (Tavtigian et al. 2020): applies when no rule-based combination fires; each tag carries a defined point value; point thresholds determine classification

PP3/BP4 computational evidence uses Pejaver et al. (2022) [13] calibrated tool-specific thresholds, not simple vote-counting.

ACMG Criteria - Full Details

Each criterion implemented in VarViz with exact trigger conditions, data sources, and mutual-exclusion rules.

◆ Pathogenic Evidence Tags

PS1 / PS1_moderate / PS1_supporting - Pathogenic (star-weighted)  PS1 PS1_mod PS1_sup  +4 / +2 / +1 pts

Trigger: ClinVar contains the exact same amino acid change, classified as Pathogenic. Strength is scaled by ClinVar review star rating:

ClinVar StarsReview StatusPS1 StrengthPoints
★★★★ / ★★★Practice guideline / Expert panelPS1 Strong+4
★★Multiple submitters, criteria providedPS1 Strong+4
★Single submitter, criteria providedPS1_moderate+2
☆No assertion criteria providedPS1_supporting+1

Rationale: A 0-star ClinVar assertion without assertion criteria carries far less evidentiary weight than a 4-star expert panel classification. Star-weighting follows GeneBe implementation (Maj et al. (2023) [18]).

Exclusions: Mutually exclusive with PM5 and BP6. If ClinVar significance is only Likely Pathogenic, PS1 is downgraded to PM5 (+2 pts). Suppressed when BA1 or BS1 fires.

Data source: NCBI ClinVar via E-utilities.

PM1 / PM1_strong - Moderate / Strong Pathogenic  PM1 PM1_strong  +2 / +4 pts

Trigger (any pathway):

  1. CCRS constraint: Variant falls in a constrained coding region (≥90th percentile CCRs score).
  2. UniProt domain: Variant is in a structurally or functionally critical annotated domain.
  3. 15 aa ClinVar neighborhood (new): ≥3 Pathogenic/LP ClinVar variants fall within ±15 amino acids of the query position AND ≥75% of all classified variants in that window are pathogenic. This identifies mutational hotspots not captured by named UniProt domains. Uses the already-fetched ClinVar dataset — no additional API call. The Mutation Density track and Density Dominant column in the annotation table provide complementary visual corroboration of this signal.

Upgrade to PM1_strong (PS-equivalent, +4 pts):

  • UniProt domain hit and strong conservation: ConSurf ≥8, PhyloP 100V >4.0, PhastCons >0.9, GERP++ >4.4, or ScoreCons >0.8.
  • UniProt domain hit and neighborhood hotspot (pathways 2+3 combined).

Not applied when: gnomAD AF > 1% (BS1/BA1 range).

Data sources: CCRS local file, UniProt features API, ClinVar (neighborhood), ConSurf upload, UCSC REST API, MyVariant.info (GERP++), CCRStoAAC (ScoreCons).

PM2 - Moderate Pathogenic  PM2  +2 pts

Trigger: gnomAD AF < 0.0001 (0.01%) or absent from gnomAD.

Data source: gnomAD GraphQL API (v4; v2 fallback).

PM3 / PM3_moderate / PM3_strong - Pathogenic (user input)  PM3 PM3_mod PM3_str  +1 / +2 / +4 pts

Trigger: User-specified compound heterozygosity evidence from clinical records or literature. Appears as a per-variant dropdown in the ACMG Classification row, labeled In trans (PM3 — AR genes). Only applicable to autosomal recessive genes.

SelectionEvidencePoints
1 compound het observationPM3 Supporting+1
2 compound het observationsPM3 Moderate+2
≥3 compound het observationsPM3 Strong+4

Evidence tiers follow ClinGen PM3 scoring recommendations. To find published compound heterozygosity reports, search PubMed for [gene] [variant] compound heterozygous. The AutoPM3 tool (Li et al., 2025 [19]) automates LLM-based extraction of PM3 evidence from scientific literature.

Exported columns: PM3_Evidence (description), PM3_Applied (tag), included in Final_ACMG_Tags and Final_Classification.

Data source: User-provided clinical data. Not automatable without phase information.

PM4 - Moderate Pathogenic  PM4  +2 pts

Trigger: Protein length change detected in variant name: contains del, ins, dup, fs, or frameshift.

PS3_supporting - Supporting Pathogenic (PTM site)  PS3_sup  +1 pt

Trigger: The variant position coincides with a UniProt-annotated post-translational modification site of high functional significance.

Strong PTM types (fire PS3_supporting +1 pt): phosphorylation, ubiquitination, acetylation, disulfide bond, cross-link, SUMOylation.

Moderate PTM types (annotated in table, no ACMG point): glycosylation, lipidation, methylation.

Rationale: Missense variants that abolish a known phosphorylation, ubiquitination, or disulfide site have well-established functional consequences. UniProt PTM annotations are reviewed and represent experimental evidence. This fires as PS3 at Supporting strength (+1 pt), consistent with ClinGen PS3/BS3 calibration guidance for gene-specific functional evidence.

The PTM_ACMG and PTM_Strength columns in the variant annotation table record the classification. PTM sites are also visible as vertical marks in the ClinVar/PTM/CCRS track.

Data source: UniProt REST API features endpoint (types: mod_res, crosslnk, disulfid, lipid, carbohyd).

PM5 - Moderate Pathogenic  PM5  +2 pts

Trigger: A different pathogenic amino acid change exists at the same codon in ClinVar (clinvar_match_type == "position"), or the exact change exists but ClinVar classifies it only as Likely Pathogenic (PS1 downgraded).

Exclusions: Mutually exclusive with PS1.

PP2 - Supporting Pathogenic  PP2  +1 pt

Trigger (any one of three):

  1. ClinVar contains ≥3 Pathogenic/Likely Pathogenic missense variants for this gene.
  2. ClinGen Gene-Disease Validity is Definitive or Strong.
  3. GeVIR percentile <25 — the gene is highly intolerant to missense variation across 141,456 gnomAD individuals (Abramovs et al. (2020) [17]), indicating missense changes are a common disease mechanism independent of ClinVar ascertainment.

Gated off when: gnomAD AF > BS1 threshold (BS1 firing suppresses PP2).

Data source: ClinVar API, ClinGen LDH/eRepo, GeVIR gene ranking table (gevirank.org).

PP3 / PP3_moderate / PP3_strong - Supporting to Strong Pathogenic  PP3 PP3_mod PP3_str  +1 / +2 / +4 pts

Multi-tier computational pathogenicity evidence following Pejaver et al. (2022) [13] calibrated thresholds. Hierarchy (highest tier wins; each tool can only upgrade, never downgrade):

PP5 - Supporting Pathogenic  PP5  +1 pt

Trigger: ClinVar exact match is Pathogenic or Likely Pathogenic (complementary supporting evidence alongside PS1/PM5).

Requires: Exact protein change match. Gated off when BA1 or BS1 fires.

PP1 - Cosegregation  PP1  +1 pt  |  PP1↑  +2 pts  |  PP1⇑  +4 pts

Set using the Cosegregation dropdown in each variant card.

  • PP1 Supporting (+1 pt): 1–2 affected relatives with variant
  • PP1 Moderate (+2 pts): 3–4 affected relatives (or LOD ≥ 1.5)
  • PP1 Strong (+4 pts): ≥5 affected relatives (or LOD ≥ 3.5)

Gated off when: BA1 or BS1 fires (variant too common to be disease-causing).

Citation: Jarvik and Browning (2016) [15].

PS2 / PM6 - De novo (per-variant)  PS2 PM6  +4 / +2 pts

PS2 (+4 pts, Strong): Confirmed de novo — both parents tested, variant absent in both, in an affected individual with no relevant family history.

PM6 (+2 pts, Moderate): Assumed de novo in an affected individual, without parental confirmation.

Input: Each variant card has its own De novo dropdown in the ACMG Classification row, independent from other variants in the panel. This allows variant-specific de novo evidence in multi-variant sessions.

Effect on PM2: Selecting de novo sets the inheritance mode to monoallelic for that card, using stricter AF thresholds for PM2.

Exported columns: DeNovo_Evidence, DeNovo_Applied, included in Final_ACMG_Tags and Final_Classification.

Data source: User-provided clinical data.

◆ Benign Evidence Tags

BA1 - Stand-alone Benign  BA1  −8 pts

Trigger: gnomAD AF > 5%.

Effect: Immediately classifies as Benign. Removes all pathogenic tags (PM1, PM2, PM4, PM5, PP2, PP3, PS1, PP5). Overrides all other evidence.

BS1 - Strong Benign  BS1  −4 pts

Trigger: gnomAD AF > 1% (up to 5%; beyond that is BA1).

Effect: Removes pathogenic moderate/supporting tags (PM1, PP2, PP3, PM5, PS1, PP5).

BS2 - Strong Benign  BS2  −4 pts

Trigger: Variant observed in homozygous state in gnomAD (nhomalt > 0), or gnomAD AF > 5% as proxy when homozygote count unavailable.

Independent of BS1/BA1 - evaluates healthy carrier observation separately.

BP1 - Supporting Benign  BP1  −1 pt

Trigger: Missense variant in a gene where population genetics evidence indicates missense variation is tolerated — GeVIR percentile >75 (gene accumulates missense variants freely in gnomAD).

This operationalizes the ACMG/AMP definition of BP1 ("missense variant in a gene for which primarily truncating variants are known to cause disease") using gene-level missense constraint data rather than a curated list, avoiding circular use of ClinVar evidence.

Suppressed when: BA1 or BS1 fires (frequency evidence supersedes mechanism evidence); PS1 or PP5 fires (ClinVar pathogenic at this exact position contradicts BP1); ClinGen Gene-Disease Validity is Definitive or Strong for this gene.

Data source: GeVIR gene ranking table (Abramovs et al., 2020 [17]) — gevirank.org.

Hover tooltip: All ACMG criterion cards show the GeVIR percentile on mouse hover, including the exact score used for PP2 and BP1 decisions.

BP3 - Supporting Benign  BP3  −1 pt

Trigger: In-frame indel (del/ins/dup in variant name, no fs) falls within a UniProt-annotated repeat region.

Data source: UniProt features API (type == “repeat” checked against position).

BP4 - Supporting Benign  BP4  −1 pt

Trigger: ≥2 benign predictor votes AND PP3 has not fired. See per-tool thresholds in PP3 section above.

Suppressed when: Any PP3 tier is active (mutual exclusion).

BP6 - Supporting Benign (BS-equivalent in Tavtigian)  BP6  −4 pts

Trigger: ClinVar exact match is Benign (not conflicting, not just Likely Benign).

Requires: Exact protein change match. Mutually exclusive with PS1 - if PS1 has fired, BP6 is blocked.

BP7 - Supporting Benign  BP7  −1 pt

Trigger: Synonymous variant (reference and alternate amino acids identical, e.g. p.Pro72Pro).

Data source: Variant name parsing.

ClinVar Matching Logic

VarViz uses a two-tier matching strategy for each input variant:

Match TypeLogicDisplayACMG Tags Eligible
Exact User’s protein change (3-letter and 1-letter) found verbatim in ClinVar variant name “Pathogenic” (clean) PS1, PP5, BP6
Position Different variant at same amino acid position in ClinVar (e.g. query p.Cys29Trp, ClinVar has p.Cys29Ter) “Pathogenic” + ⚠ diff variant at pos PM5 only
None No ClinVar entry at this position “Not in ClinVar” None
Why this matters: A stop-gain (p.Cys29Ter) and a missense (p.Cys29Trp) have different molecular consequences. PS1 requires the identical amino acid change. VarViz correctly assigns PM5 for position matches and PS1 only for exact amino acid change matches - preventing inflated pathogenicity calls.

Controls & Filters

Variant Input

Comma-separated HGVS protein notation: p.R175H, p.G245S, p.V272M. Both 1-letter and 3-letter forms accepted. Variants appear as dashed lines on all tracks and color-coded lollipops on the domain bar.

Output Tracks

  • gnomAD Frequency, Mutation Density, AF Mean Pathogenicity, ClinVar / PTM / CCRS: core tracks, always available
  • Multi Conservation (UCSC): PhyloP 100V, PhyloP 470M, PhastCons; fetched live from UCSC REST API
  • Conservation (ConSurf): shown automatically when a ConSurf grades file is uploaded

ClinVar Variants Filter

  • Any Star: all assertions
  • 1 Star or more: criteria provided, single submitter
  • 2 Stars or more: multiple submitters, no conflicts

Frequency Cutoff Method

  • Calculate AF (by prevalence): maximum credible AF from disease prevalence, heterogeneity, penetrance (Whiffin et al. (2017) [14]). Adjust via Edit parameters. Default: prevalence 1 in 2,000; allelic heterogeneity 0.2; penetrance 0.5.
  • AC Filter (by max AF): set maximum allele frequency directly
Note: Max AF / Max AC updates live but the plot only re-renders on Go. Click Go again if numbers appear inconsistent with the plot.

Optional File Uploads

  • ConSurf grades file: consurf.grades from ConSurf-DB. Upload before clicking Go.
  • Custom pathogenic variants: protein positions for the green density overlay.

Analysis Parameters

The sidebar parameters govern how allele frequency thresholds are derived and which inheritance model is applied. All values are recorded in the TSV export for reproducibility.

Inheritance Mode

Select Monoallelic (autosomal dominant, X-linked) or Biallelic (autosomal recessive) in the sidebar. This controls two thresholds:

  • PM2 threshold: monoallelic = prevalence-derived AF (typically <0.0001); biallelic = 0.01
  • BS1 threshold: monoallelic = 0.01; biallelic = 0.05

The actual thresholds used for each variant are recorded in Analysis_PM2_Threshold and Analysis_BS1_Threshold in the TSV.

Frequency Cutoff (PM2 AF threshold)

Two methods are available via the Edit parameters panel:

  1. Calculate AF (by prevalence): derives maximum credible allele frequency from disease prevalence, allelic heterogeneity, genetic heterogeneity, and penetrance using the Whiffin et al. (2017) [14] formula. Parameters are stored in Analysis_Prevalence, Analysis_Allelic_Het, Analysis_Genetic_Het, Analysis_Penetrance.
  2. Manual AC filter: directly specify maximum tolerated allele count from population size and confidence interval. Stored in Analysis_AC_Cutoff, Analysis_Pop_Size, Analysis_CI.

The selected cutoff method is recorded in Analysis_Cutoff_Method.

Analysis Parameters in Variant Cards

A ⚙ Analysis parameters row appears in each variant card between the annotation data and the ACMG Classification section. It displays colour-coded pills showing the active inheritance mode, PM2 AF threshold, disease prevalence, ClinGen gene-disease classification, and ConSurf file name (if uploaded). This makes the analytical context visible at a glance without requiring the user to scroll back to the sidebar.

Caching

All API responses and local lookups are cached in memory for the session. Returning to a previously loaded gene is instant. Cached sources: UniProt gene info, UniProt features, AlphaFold pLDDT, AlphaFold pathogenicity, gnomAD API, gnomAD parsed data, ClinVar, CCRS regions, dbNSFP (MyVariant.info), NCBI gene summaries, UCSC conservation (per transcript), Ensembl exon structure.

Failed results are not cached. Click Go again to retry.

Console: [Cache hit] messages appear when cached data is reused. Step-by-step API fetch messages appear on first load.

FAQ

Why “No gnomAD data”?

The gnomAD API may be temporarily overloaded. VarViz retries up to 3 times and falls back from v4 to v2.1.1. Click Go again - failed results are never cached.

Multi Conservation track is empty or slow?

UCSC and Ensembl REST APIs are queried live. First load fetches exon structure from Ensembl then PhyloP/PhastCons for each exon from UCSC - can take 5–15 s for large genes. Results are cached for the session. Unusual canonical transcripts or UCSC API unavailability may cause empty results - click Go to retry.

ClinVar track is empty?

The gene may lack pathogenic/LP assertions in ClinVar, or may only have LoF variants. Try the “Any Star” filter.

My gene isn’t in the dropdown?

VarViz includes 19,044 protein-coding genes from GENCODE v49 with reviewed UniProt entries. Non-coding genes, pseudogenes, and unreviewed entries are excluded. Check if the gene was recently renamed by HGNC.

How are ACMG tags computed?

VarViz uses a hybrid engine: Richards 2015 rule-based classification fires first; if no rule matches, the Tavtigian 2020 Bayesian point system applies. PP3/BP4 use Pejaver et al. (2022) [13] calibrated predictor thresholds. PS1/BP6 require exact ClinVar protein change matches; PM5 fires for position-only matches. See ACMG Overview and Criteria Details. Always review tags with clinical context.

What does PP3_strong / PP3_moderate mean?

PP3 has three tiers with different point values. PP3_strong (+4 pts, PS-equivalent) = REVEL ≥ 0.932 only. PP3_moderate (+2 pts, PM-equivalent) = REVEL ≥ 0.773, or CADD ≥ 35, or ≥2 meta-predictors + calibrated corroboration, or ≥2 strong conservation hits + calibrated predictor. PP3 supporting (+1 pt) = standard supporting level. Displayed in the table as PP3 (supporting), PP3↑ (moderate), PP3⇑ (strong).

What does PM1_strong mean?

PM1_strong is a PS-equivalent upgrade (+4 pts instead of +2). It fires when the variant is in a structurally critical UniProt domain and at least one strong conservation signal is present (ConSurf ≥8, PhyloP >4.0, PhastCons >0.9, GERP++ >4.4, or ScoreCons >0.8). Domain location + evolutionary conservation together provide near-PS-level evidence.

ClinVar says “⚠ diff variant at pos” - what does that mean?

ClinVar has a pathogenic variant at the same amino acid position but a different protein change than your query. PS1 requires identical amino acid changes. VarViz correctly assigns PM5 for position matches and PS1 only for exact amino acid change matches.

Why are dbNSFP scores showing “-”?

dbNSFP covers most coding missense variants but may lack data for rare or novel positions. VarViz queries MyVariant.info with both 3-letter and 1-letter protein notation. Synonymous, nonsense, and indel variants are not covered by most dbNSFP tools. ACMG PP3/BP4 will still compute from whatever scores are available.

How do I upload ConSurf scores?

Visit ConSurf-DB, search your protein, download consurf.grades, and upload using the file input on the left panel before clicking Go. The Conservation track appears automatically when a file is uploaded - no checkbox selection needed.

How do the per-variant clinical dropdowns work?

Each variant card has three independent dropdowns in the ACMG Classification row: De novo (PS2/PM6), Cosegregation (PP1), and In trans / compound het (PM3). These allow clinical evidence to be applied per-variant within a multi-variant panel. Choose the number of affected family members carrying the variant. PP1 is applied at Supporting (+1 pt) for 1-2 relatives, Moderate (+2 pts) for 3-4, and Strong (+4 pts) for 5 or more. The classification updates when you change the selection. Results are included in the TSV download. PP1 is not applied when BA1 or BS1 is present.

How do I reset the zoom?

(1) Double-click anywhere on the plot. (2) Click the house icon (Reset axes) in the plotly toolbar. (3) Click Autoscale.
